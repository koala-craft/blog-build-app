# フロントエンド実践ステップ (Next.js App Router)

本ドキュメントはアプリケーションのUI部分（ダッシュボード画面群）を細かな単位に分割し、段階的に組み上げるための手順書です。洗練されたモダンなデザイン（ガラスモーフィズムやスムーズなアニメーション）を意識して構築します。

---

## 🎨 Step 1: 初期設定とデザインシステム基盤
- [x] **1.1 設定ファイル群** - `frontend/.env.local` に `NEXT_PUBLIC_API_URL` を設定。
- [x] **1.2 スタイリング基盤** - `tailwind.config.ts` や `src/app/globals.css` を編集し、プレミムなカラーパレット（ダークモード、グラデーション）やカスタムフォントを設定。
- [x] **1.3 APIクライアント** - `src/lib/api.ts` を作成し、バックエンド（FastAPI: 8000ポート等）と通信するための共通 fetch ヘルパー関数を実装。

## 🧩 Step 2: 共通コンポーネントとレイアウト
- [x] **2.1 UI部品群** - `src/components/ui/` 配下に Button, Card, Input, Modal, Table などの再利用可能なモダンコンポーネントを作成。
- [x] **2.2 ナビゲーション** - `src/components/layout/Sidebar.tsx` を作成し、各機能への左側メニュー（`lucide-react` アイコン付き）を実装。
- [x] **2.3 ルートレイアウト** - `src/app/layout.tsx` を編集し、Sidebarとメインコンテンツエリアを配置。

## 📊 Step 3: ホーム（分析・ダッシュボード機能）
- [x] **3.1 分析画面** - `src/app/page.tsx` において、`/api/analysis/` に GETリクエストを送り、高評価のナレッジランキングや生成記事数を表示。
- [x] **3.2 グラフ・指標ビュー** - データを視覚的に「自己進化の進度」として美しく見せる KPI カード等の配置。

## 📡 Step 4: 収集（Collect）＆ 抽出（Extract）UI
- [x] **4.1 収集画面 (`/collect`)** - ハッシュタグ名などを入力し、noteからの悩み収集（`/api/collect/note`）をキック・結果を表示する画面。
- [x] **4.2 抽出画面 (`/extract`)** - 手動で読書メモ等のフリーテキストを入力し、`/api/extract` へ投げて抽出されたJSONデータをプレビュー・修正し、そのまま知識として保存ボタンを押せる画面。

## 📚 Step 5: ナレッジデータベース管理画面 (`/knowledge`)
- [x] **5.1 カテゴリタブ構築** - 「占星術」「マインドフルネス」「ペルソナ」「コピー型」など7つの柱を切り替えるタブUIを作成。
- [x] **5.2 一覧＆追加モーダル** - 選択タブごとに `/api/knowledge/${category}` を叩いて一覧を表示し、新規作成（追加）時は各カテゴリに合わせたフォームを表示＋保存処理。

## 💡 Step 6: お題生成（Topics）UI (`/generate`)
- [x] **6.1 お題リスト画面** - 未使用の悩みから生成された `blog_topics_candidates` を一覧表示する画面。
- [x] **6.2 お題自動生成ボタン** - `/api/topics/generate` を叩いて複数のお題を非同期で補充するアクション。
- [x] **6.3 記事執筆キック** - リスト横にある「記事を書く」ボタンから、記事生成画面（ストリーミング画面）へ遷移する処理。

## ✍️ Step 7: 記事リアルタイム生成＆管理画面 (`/articles`)
- [x] **7.1 記事一覧・フィードバック画面** - 過去に生成された記事のリスト、ステータス、そして「スキ」「コメント数」等を手動入力して擬似的にフィードバック（PATCH `/api/articles/{id}/feedback`）する機能。
- [x] **7.2 記事生成（ストリーミング）画面** - `src/app/articles/stream/[topic_id]/page.tsx`。SSE（Server-Sent Events）を用いて、LLMがタイピングアニメーションのようにリアルタイムで記事を返してくる様子を `react-markdown` で描画。
- [x] **7.3 再生成ボタン** - 気に入らない記事を `/api/articles/{id}/regenerate` で再度ストリーミングを走らせるUI機能の実装。

---

🎉 **Frontend Implementation Completed!**
全ての画面とバックエンドAPIの接続、そしてプレミアムなガラスモーフィズムUIの構築が完了しました。
記事生成のストリーミング表示から、反響フィードバックによる自己進化の可視化まで全機能が動作します。
